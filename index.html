Slack - https://slack.com/

koristimo za

  • brzu i direktnu komunikaciju
  • notifikacije iz produkcije
  • direktnu suradnju u rješavanju problema

BaseCamp - https://basecamp.com/

namjenjen je za sve projetke i produkte sadrži

  • Opise projekta
  • Dokumentaciju za projekt
  • Zadatke u projektu
BaseCamp je alat za suradnju a manje za pregled stausa projekta projekti na basecamp-u žive kraće od produkata

GitHub https://github.com/

je centar svemira za developere i sadrži

  • source code od svih servisa i aplikacija
  • source code infrastrukture (infrastructure as code)
  • opise servisa i komponenti
  • omogućuje suradnju unutar i među timovima
  • timovi koje imamo su preslikani u GitHub timove

Razvojna okolina

Ako svi imaju istu razvojnu okolinu, onda možemo

  • brzo uključiti nove ljude
  • spojiti različite timove i olakšati suradnju

Naša razvojna okolina je par alata i skripte koje povlače source i omogućuju lako pokretanje

Smjernice za developere

Svaki servis i aplikacija se mora

  • moći pustiti u produkciju u bilo kojem trenutku
  • moći vratiti na prethodnu verziju u slučaju problema
  • imati automatizirane testove

Ove smjerince zahtjevaju pažljivo planiranje razvoja

  • koliko traje pokretanje aplikacije
  • kako ne kreirati zastoje
  • kako modelirati podatke i komunikaciju
  • kako znamo da ne radi dobro ili postoji problem

Što savjetovati mlađim ljudima?

  • Koristie neki SCM (Git, Svn, Marcurial...)
  • Code koji pišete, pišete za ljude
  • Pišite testove za code
  • "Posudite" od najboljih

Još malo o SCM-u

Nikad ne radite commit ako se code ne compile-a

Nastojite raditi commit barem jednom dnevno

Test coverage ne smije pasti sa commitom

Code koji pišete, pišete za ljude

Vaš code će najviše čitati i pokušati razumjeti ljudi

Vodite računa o nazivima varijabli, modula, funkcija...

Nemojte nikad biti zadovoljni, uvijek može bolje

Gotovo svaki komadić ćete napisati i barem 3x promijeniti

Pišite testove za code

Naknade promjene su najčešći uzrok bugova,
